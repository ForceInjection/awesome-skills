# 网页内容翻译器 (Web Content Translator)

这个技能用于将指定的网页内容下载并转换为 Markdown 格式，同时翻译成目标语言（默认中文），并自动提取、下载网页中的图片到本地的 `img` 目录中，最后将图片重命名为有意义的名称并在 Markdown 中正确引用。

## 工作流 (Workflow)

严格按照以下步骤执行：

### 1. 网页内容获取与转换

- 优先使用 Jina Reader 获取内容：使用终端命令 `curl -s "https://r.jina.ai/<目标URL>"`，它会直接返回排版良好的 Markdown 格式正文。
- 如果 Jina Reader 获取失败或返回为空，再回退使用其他工具（如 `mcp_DuckDuckGo_Search_Server_fetch_content`、`curl` 或等效工具）获取目标 URL 的页面内容并将 HTML 清理转换为 Markdown。
- 如果用户指定了翻译（或默认翻译为中文），请在转换过程中将文本内容进行专业、严谨的翻译。
- 如果原网页中包含表格（HTML `<table>`），必须将其准确地转换为标准的 Markdown 表格格式（`|---|---|`），并注意避免在 Markdown 表格内使用 `<br>` 等 HTML 换行符号，保持表格的整洁性。
- 确保排版符合规范（例如：中英文之间留有空格、代码块有注释说明等）。

### 2. 图片链接提取

- 解析原网页内容或 HTML，提取出所有核心配图的链接（`<img src="...">`）。
- 过滤掉无关的图标 (favicon)、追踪像素等无意义的图片。

### 3. 创建目录与下载图片

- 检查当前工作区是否存在 `img` 目录，如果不存在，则使用终端命令 `mkdir -p img` 创建。
- 使用终端命令（如 `curl -O` 等）将提取到的图片批量下载到 `img` 目录中。

### 4. 图片重命名

- 根据文章的上下文或图片所表达的内容，为下载的图片生成**有意义的英文或拼音名称**（例如：`design-patterns.png`）。
- 使用终端命令（如 `mv`）对下载好的图片进行重命名。

### 5. 更新 Markdown 引用

- 将 Markdown 文档中原来的远程图片链接替换为本地图片链接（例如：`![design-patterns](./img/design-patterns.png)`）。
- 将最终的 Markdown 文本保存到用户指定的文件中，或者根据网页标题生成一个对应的 `.md` 文件并保存。
